**Project:** SILENCE.OBJECTS  
**Module:** @silence/engine  
**Version:** 0.1.0-mvp  
**Date:** 2026-06-10  
**Owner:** security-lead  
**Classification:** RESTRICTED

---

## 1. PURPOSE

This policy defines the generation, storage, rotation, and revocation procedures for cryptographic keys used to sign and verify `@silence/engine` artifacts (native binaries and WASM modules).

---

## 2. KEY HIERARCHY

| Key Type | Algorithm | Usage | Lifecycle |
|----------|-----------|-------|-----------|
| Signing key (private) | Ed25519 | Sign engine artifacts | 90 days |
| Verification key (public) | Ed25519 | Verify signatures in client apps | Permanent (rotated with signing key) |
| CI signing key | Ed25519 | Staging builds in GitHub Actions | 90 days |
| Production signing key | Ed25519 | Production releases via HSM | 90 days |

---

## 3. GENERATION

### 3.1 Dev / Local

```bash
openssl genpkey -algorithm Ed25519 -out engine-signing-key.pem
openssl pkey -in engine-signing-key.pem -pubout -out engine-signing-key.pub
```

- Stored in `01_governance/RELEASE_KEYS/`
- `.gitignore` MUST exclude `*.pem` and `*.pub` in this directory
- Auto-generated by `scripts/sign-engine.sh` if absent (dev convenience only)

### 3.2 Staging

- Generated by CI workflow (`engine-build.yml`)
- Stored as GitHub Secret `ENGINE_SIGNING_KEY_PEM`
- Public key exported as artifact `engine-signing-key.pub`

### 3.3 Production

- Generated inside HSM (YubiHSM 2 or AWS CloudHSM)
- Private key NEVER leaves HSM
- Public key exported manually and pinned in client apps

---

## 4. STORAGE

| Environment | Storage Medium | Access Control | Backup |
|-------------|---------------|----------------|--------|
| Dev | Local filesystem (encrypted volume) | Developer workstation | None (disposable) |
| Staging | GitHub Secrets + CI artifact | Repository admin | GitHub secret recovery |
| Production | HSM | ops-lead + security-lead | HSM backup procedure |

---

## 5. ROTATION

### 5.1 Schedule

- **Dev:** Per CI run (ephemeral)
- **Staging:** 90 days
- **Production:** 90 days

### 5.2 Rotation Procedure

1. Generate new key pair
2. Sign a new engine artifact with the new key
3. Update public key in all client apps (`05_apps/*/public/engine-signing-key.pub`)
4. Update `ENGINE_SIGNING_KEY_PEM` GitHub Secret (staging)
5. Rotate HSM key slot (production)
6. Publish new public key hash in `EFFECTLOG.md`
7. Revoke old key (mark as expired in key inventory)

### 5.3 Key Inventory

```
01_governance/RELEASE_KEYS/
├── key-inventory.json        — SSoT for all active keys
├── engine-signing-key.pem    — CURRENT dev key
├── engine-signing-key.pub    — CURRENT dev public key
└── revoked/
    ├── engine-signing-key-2026-Q1.pem
    └── engine-signing-key-2026-Q1.pub
```

---

## 6. REVOCATION

A key MUST be revoked immediately if:
- The private key is suspected to be compromised
- A developer with access leaves the team
- The CI secret is exposed in logs

**Revocation steps:**
1. Mark key as `REVOKED` in `key-inventory.json`
2. Move key files to `revoked/` directory
3. Update client apps to reject signatures from revoked key
4. File `EFFECTLOG.md` entry with incident description

---

## 7. VERIFICATION IN CLIENT APPS

```typescript
// Pseudo-code for WASM load-time verification
async function verifyEngineWasm(wasmBytes: ArrayBuffer, pubKeyPem: string, sigB64: string): Promise<boolean> {
  const hash = await crypto.subtle.digest('SHA-256', wasmBytes);
  // Verify Ed25519 signature using Web Crypto or @noble/ed25519
  return ed25519.verify(sigB64, hash, pubKeyPem);
}
```

- Verification is OPTIONAL in dev (`SILENCE_ENGINE_VERIFY_MODE=skip`)
- Verification is MANDATORY in staging and production (`SILENCE_ENGINE_VERIFY_MODE=verify`)
- Failure to verify MUST block engine initialization
