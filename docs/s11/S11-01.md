# S11-01 — S11 GOVERNANCE BASELINE

## 1. Cel

Niniejszy dokument stanowi SSoT dla kontroli S11 (terminologicznej i behavioralnej) w całym workspace SILENCE.OBJECTS. Definiuje klasy zabronionych terminów, dozwolone alternatywy, architekturę enforcement (G1/G2) oraz metryki coverage.

## 2. Scope

S11 obejmuje wszystkie artefakty w workspace:
- Kod źródłowy (TypeScript, JavaScript, Rust, itp.)
- Dokumentację governance (01_governance/, docs/)
- Protokoły (02_protocols/)
- Aplikacje (05_apps/)
- UI, prompty, kontrakty UX

## 3. Current Baseline

Na dzień aktualizacji `@silence/s11-lint` v2.1.0 jest aktywnym pakietem workspace, a `s11-check` jest działającym wrapperem CI, co oznacza, że G1 jest WDROŻONE jako merge-blocking gate.

Enforcement G1 obejmuje obecnie katalogi `01_governance`, `02_protocols`, `04_packages` oraz `05_apps`, zgodnie z konfiguracją rootowego `package.json`.

Dla przejrzystości S11 rozróżnia status G1 i G2 (kontekst: językowy / terminologiczny):

- **G1 (buildtime linter):** ACTIVE — `@silence/s11-lint` jest produkcyjnym pakietem workspace, CLI `s11-lint` jest wywoływany przez `s11-check` i może blokować merge w CI.
- **G2 (runtime S11 guardrails):** DESIGN / POC — edge middleware filtrujący input/output promptów nie jest jeszcze globalnie wdrożony; status pozostaje PARTIAL do czasu pełnej implementacji.

**Uwaga dotycząca nomenklatury:** W dokumentach filarowych (FIVE-PILLARS, REUSE-AND-GAP-ANALYSIS) występuje osobne oznaczenie „G2” dotyczące metering pipeline + rich 402 response (Filar I/IV). Nie jest to ten sam G2 co S11 runtime guardrails. W kontekście S11-01 „G2” oznacza wyłącznie językowy runtime filter; w kontekście delivery planu „G2 (Metering 402)” oznacza komponent billingowy.

Coverage G1 jest funkcją realnej zawartości repo: dopóki `04_packages` i `05_apps` zawierają głównie stuby, liczba realnie kontrolowanych plików jest ograniczona, dlatego G1 musi być traktowane jako living control, którego zasięg rośnie wraz z rozbudową kodu.

## 5. Enforcement Architecture

### G1 — Buildtime Linter

- Pakiet: `04_packages/@silence/s11-lint`
- CLI: `s11-lint`
- Wrapper CI: `02_protocols/s11-check.js`
- Root command: `pnpm s11-check`
- Exit code: 1 on violations (unless `--report-only`)

### G2 — Runtime Guardrails

- Target: edge middleware (Vercel Edge / Supabase Edge Functions)
- Scope: input/output filtering dla promptów i response
- Status: PARTIAL / DESIGN

## 6. Metryki Coverage

Monitorowane per release w EFFECTLOG:
- `G1_FILES_SCANNED` — liczba plików faktycznie przeskanowanych
- `G1_WARNINGS` — liczba naruszeń (powinna być 0 przed merge)
- `G1_COVERAGE_RATIO` — udział plików z kluczowych domen objętych G1

Target PCS: > 0.99
